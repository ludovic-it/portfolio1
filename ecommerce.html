SIO-SHOES

Site e-commerce de vente de chaussures développé avec Symfony

Démonstration

Présentation du projet

Dans le cadre de ma formation en BTS SIO option SLAM, j'ai développé un site e-commerce SIO-SHOES de vente de chaussures à l'aide du framework Symfony. L'objectif était de concevoir un site web e-commerce permettant aux utilisateurs de naviguer parmi différents produits, de créer un compte, de gérer leur panier, de passer commande et d'effectuer un paiement en ligne.

Le site devait également offrir aux administrateurs la possibilité de gérer les produits, les commandes et les frais de livraison par ville, tout en maintenant une interface claire et sécurisée.

Fonctionnalités principales

Le site affiche les produits classés par catégories et sous-catégories, avec un système de pagination pour faciliter la navigation. Les utilisateurs peuvent créer un compte, se connecter et accéder à leur panier pour ajouter, modifier ou supprimer des articles.

Les rôles définis (USER, EDITOR, ADMIN) permettent de restreindre ou d'autoriser certaines actions, comme la gestion des produits ou des utilisateurs.

Les commandes validées sont enregistrées en base de données et un e-mail de confirmation est envoyé automatiquement, avec un PDF généré pour le bon de commande.

Base de données : my_shop_dev

Les rôles

01

Users

Ils peuvent naviguer sur le site, consulter les différentes catégories, ajouter des articles à leur panier, procéder aux paiements, s'inscrire et se déconnecter.

02

Editor

Ils ont tous les droits des utilisateurs mais peuvent aussi ajouter, supprimer des produits, des catégories et des villes. Accès à l'interface qui affiche les commandes de tous les utilisateurs.

03

Admin

Ils ont tous les droits des utilisateurs et des éditeurs mais peuvent aussi gérer tous les utilisateurs de la base de données, les supprimer ou leur attribuer des rôles.

La création des entités et les migrations

J'ai utilisé symfony console make:entity Product pour créer une entité dans Symfony. Cette entité correspond à la table Product à laquelle on peut attribuer des champs (id, name, price, image et stock), ce qui permet à Symfony de générer automatiquement les méthodes get et set.

symfony console make:migration génère un fichier de migration qui contient toutes les requêtes SQL pour créer ou modifier la table product, mais elle n'est pas encore envoyée dans la base, elle prépare juste le script.

symfony console doctrine:migration:migrate applique les migrations à la base de données en exécutant le fichier généré par la commande précédente.

Architecture

Le projet utilise l'architecture MVC de Symfony pour organiser le code et séparer la logique métier, les données et la vue. Les entités Doctrine permettent de gérer les données et les relations avec la base MySQL, tandis que Twig et Bootstrap sont utilisés pour l'affichage des pages et le design responsive. Le composant Form de Symfony est utilisé pour la gestion des formulaires.

Gestion des routes

Le site utilise le système de route de Symfony pour associer chaque page à une URL. Les routes permettent de définir à quelles pages (URL) un utilisateur va pouvoir accéder. Certaines routes utilisent des paramètres dynamiques, comme l'identifiant d'un produit ou d'une commande, et l'accès à certaines pages est restreint selon les rôles utilisateurs.

Par exemple, l'URL /editor/{id} permet d'afficher le détail d'un produit spécifique, tandis que /product/new est réservée à la création de nouveaux produits par les administrateurs ou éditeurs. 

#[Route('/{id}/edit', name: 'app_product_edit', methods: ['GET', 'POST'])]
    public function edit(Request $request, Product $product, EntityManagerInterface $entityManager): Response
    {
        $this->checkAccess();
        $form = $this->createForm(ProductType::class, $product);
        $form->handleRequest($request);
        if ($form->isSubmitted() && $form->isValid()) {
            $entityManager->flush();
            $this->addFlash('success', 'Produit modifié avec succès.');
            return $this->redirectToRoute('app_product_index', [], Response::HTTP_SEE_OTHER);
        }
        return $this->render('product/edit.html.twig', [
            'product' => $product,
            'form' => $form,
        ]);
    }

Développement

Contrôleur Produit et gestion des stocks

Le ProductController gère toutes les opérations liées aux produits, y compris l'affichage de la liste, la création, la modification, la suppression, et la consultation détaillée d'un produit. Pour sécuriser l'accès aux fonctionnalités sensibles, j'ai mis en place une vérification des rôles : seuls les utilisateurs avec le rôle ADMIN ou EDITOR peuvent effectuer ces actions.

La création et la modification des produits passent par des formulaires Symfony. Une fonctionnalité spécifique permet également d'ajouter du stock à un produit existant, tout en mettant à jour l'historique des ajouts.

Formulaires Symfony

J'utilise les formulaires pour gérer l'inscription des utilisateurs, l'ajout de produits, la saisie des commandes et la gestion des catégories ou des villes. Chaque formulaire est lié à l'entité correspondante.

Par exemple, le formulaire CategoryType permet de créer ou modifier des catégories avec un champ pour le nom, lié directement à l'entité Category.

Gestion du panier et des commandes

Mon panier fonctionne de la manière suivante : chaque utilisateur authentifié peut ajouter des produits à son panier, peut modifier les quantités, peut supprimer des articles et peut finaliser sa commande.

Les données de l'utilisateur sont stockées dans une session Symfony ce qui lui permet de conserver son panier même s'il change de page et qu'il navigue sur le site jusqu'à sa déconnexion.

Les utilisateurs avec les rôles ADMIN et EDITOR ont accès à une page qui répertorie toutes les commandes qui ont été passées par tous les utilisateurs avec les informations client et ils peuvent valider si la livraison a été effectuée.

Le moteur de recherche

J'ai mis en place dans ma barre de navigation la possibilité d'effectuer des recherches. Par exemple : Si j'écris "nike" dans la barre de recherche, tous les éléments qui comportent le mot "nike" vont s'afficher à l'écran.

La route /search/engine va gérer le moteur de recherche quand l'utilisateur va taper un mot dans la barre de recherche. Elle va stocker le mot tapé par l'utilisateur puis elle va chercher tous les produits dont le nom correspond à celui que l'utilisateur a tapé. Les résultats sont ensuite paginés donc affichés sur plusieurs pages si nécessaire. 

#[Route('/search/engine', name: 'app_search_engine')]    
   
        public function index(ProductRepository $productRepository, CategoryRepository     
        $categoryRepository, PaginatorInterface $paginator, Request $request): Response    
    {        
        if ($request -> isMethod('GET')){            
            $data = $request->query->all();            
            $word = $data['word'];                    
            $data = $productRepository->searchEngine($word);            
            $products = $paginator->paginate(                
                $data,                
                $request->query->getInt('page',1),            
            );        
        }        
        return $this->render('search_engine/index.html.twig', [            
            'products' => $products,            
            'categories' => $categoryRepository ->findAll(),        
        ]);
    }

Outils utilisés

Le projet a été réalisé avec Symfony pour le back-end, Twig pour les templates et Bootstrap pour le front-end responsive. La base de données MySQL a été gérée avec Doctrine ORM. Stripe a été utilisé pour le paiement en ligne, smtp4dev pour tester l'envoi d'e-mails, Dompdf pour générer automatiquement les PDF des commandes, et GitHub pour stocker mes fichiers.

Compétences techniques mises en œuvre
