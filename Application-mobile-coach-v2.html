Coach IMG – Version 2

Évolution majeure avec persistance des données et architecture en couches

Nouveautés de la Version 2

Cette seconde version de l'application Coach IMG apporte des améliorations significatives par rapport à la version 1. Les principales évolutions concernent la persistance des données utilisateur et une refonte de l'architecture du projet pour une meilleure maintenabilité.

Persistance des données

Sauvegarde automatique du profil utilisateur en JSON pour retrouver ses données à chaque utilisation

Architecture en couches

Séparation en deux projets distincts : mobile (UI) et mobile.Core (logique métier)

Restauration automatique

Les champs sont préremplis au démarrage avec les dernières données saisies

Code amélioré

Classe Profil sérialisable et utilitaire Serializer dédié

Nouvelle architecture en couches

Contrairement à la V1 qui regroupait tout le code dans un seul projet, la V2 adopte une architecture en couches avec deux projets distincts au sein de la solution. Cette séparation améliore la réutilisabilité et la testabilité du code.

Avantages de cette architecture

01

Séparation des responsabilités

Le projet mobile gère uniquement l'interface XAML et les événements. Le projet mobile.Core contient toute la logique métier indépendante de l'UI.

02

Réutilisabilité

Le projet Core peut être référencé par d'autres applications (console, web, desktop) sans modification, permettant de partager la logique de calcul IMG.

03

Testabilité

Le code métier isolé dans mobile.Core peut être testé unitairement sans dépendance à l'interface utilisateur MAUI.

Persistance des données avec JSON

La fonctionnalité majeure de cette V2 est la persistance des données utilisateur. Contrairement à la V1 où les données étaient perdues à chaque fermeture de l'application, la V2 sauvegarde automatiquement le profil dans un fichier JSON local.

Les données du profil sont converties en JSON et sauvegardées dans le stockage local

Classe Profil sérialisable

La classe Profil a été enrichie avec l'attribut [Serializable] et des propriétés publiques pour permettre la sérialisation/désérialisation JSON : 

[Serializable]
public class Profil
{
    private int sexe;
    private double poids;
    private int taille;
    private int age;
    private double img;
    private string message;

    // Propriétés publiques pour la sérialisation JSON
    public int Sexe { get => sexe; set => sexe = value; }
    public double Poids { get => poids; set => poids = value; }
    public int Taille { get => taille; set => taille = value; }
    public int Age { get => age; set => age = value; }
    public double Img { get => img; set => img = value; }
    public string Message { get => message; set => message = value; }

    // Constructeur par défaut requis pour la désérialisation
    public Profil() { }

    public Profil(int sexe, double poids, int tailleCm, int age)
    {
        this.sexe = sexe;
        this.poids = poids;
        this.taille = tailleCm;
        this.age = age;
        this.img = 0;
        this.message = "";
        CalculIMG();
        ResultatIMG();
    }
}

Classe utilitaire Serializer

Une nouvelle classe Serializer a été créée pour gérer la lecture et l'écriture des données JSON. Elle utilise System.Text.Json et stocke les données dans le dossier d'application local de l'appareil. 

public class Serializer
{
    /// <summary>
    /// Sauvegarde un profil dans un fichier JSON local
    /// </summary>
    public static void serialize(string nomFichier, Profil profil)
    {
        string fichier = Path.Combine(
            Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData), 
            nomFichier);

        if (File.Exists(fichier))
        {
            File.Delete(fichier);
        }
        try
        {
            string jsonString = JsonSerializer.Serialize(profil);
            File.WriteAllText(fichier, jsonString);
        }
        catch (Exception ex)
        {
            System.Diagnostics.Debug.WriteLine("Erreur lors de la sérialisation: " + ex.Message);
        }
    }

    /// <summary>
    /// Charge un profil depuis un fichier JSON local
    /// </summary>
    public static Profil deserialize(string nomFichier)
    {
        Profil profil = null;
        string fichier = Path.Combine(
            Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData), 
            nomFichier);

        try
        {
            if (File.Exists(fichier))
            {
                string jsonString = File.ReadAllText(fichier);
                profil = JsonSerializer.Deserialize<Profil>(jsonString);
            }
        }
        catch (Exception ex)
        {
            System.Diagnostics.Debug.WriteLine("Erreur lors de la désérialisation: " + ex.Message);
        }
        return profil;
    }
}

Le fichier JSON est stocké dans le dossier d'application local, dont le chemin varie selon la plateforme :

Intégration dans l'interface

Le code-behind de MainPage.xaml.cs a été modifié pour intégrer la persistance des données. La sauvegarde s'effectue automatiquement après chaque calcul, et la restauration se fait au démarrage de l'application.

Restauration au démarrage

Une nouvelle méthode RecupProfil() est appelée dans le constructeur pour restaurer les données sauvegardées : 

private const string nomFichier = "profil.json";

public MainPage()
{
    InitializeComponent();
    RecupProfil(); // Restauration des données au démarrage
}

/// <summary>
/// Récupère le profil sauvegardé et pré-remplit les champs
/// </summary>
private void RecupProfil()
{
    Profil profil = Serializer.deserialize(nomFichier);
    if (profil != null)
    {
        // Pré-remplissage des champs avec les données sauvegardées
        entPoids.Text = profil.Poids.ToString();
        entTaille.Text = profil.Taille.ToString();
        entAge.Text = profil.Age.ToString();
        
        // Restauration du sexe sélectionné
        if (profil.Sexe == 1)
            RbHomme.IsChecked = true;
        else
            RbFemme.IsChecked = true;

        // Affichage du dernier résultat
        LblResultat.Text = $"Votre indice de masse grasse est de {profil.Img:F2} %.\n{profil.Message}";
        
        // Mise à jour du smiley
        UpdateSmiley(profil.Message);
    }
}

Sauvegarde après calcul

La méthode de calcul sauvegarde automatiquement le profil après chaque calcul réussi : 

private void BtCalculer_Clicked(object sender, EventArgs e)
{
    try
    {
        var poids = int.Parse(entPoids.Text);
        var taille = int.Parse(entTaille.Text);
        var age = int.Parse(entAge.Text);
        var sexe = RbHomme.IsChecked ? 1 : 0;

        Profil unProfil = new Profil(sexe, poids, taille, age);
        
        // NOUVEAU V2 : Sauvegarde automatique du profil
        Serializer.serialize(nomFichier, unProfil);

        LblResultat.Text = $"Votre indice de masse grasse est de {unProfil.Img:F2} %.\n{unProfil.Message}";
        UpdateSmiley(unProfil.Message);
    }
    catch (Exception ex)
    {
        DisplayAlertAsync("Erreur", "Saisie incorrecte", "Ok");
    }
}

private void UpdateSmiley(string message)
{
    if (message.Contains("Trop maigre") || message.Contains("maigre"))
        img.Source = "smiley_tropmaigre";
    else if (message.Contains("Parfait"))
        img.Source = "smiley_parfait";
    else if (message.Contains("Surpoids"))
        img.Source = "smiley_surpoids";
}

Les champs sont automatiquement préremplis au démarrage avec les dernières données saisies
