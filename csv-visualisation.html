Visualisation CSV avec JavaScript

Projet de visualisation de données CSV avec fetch() et Chart.js pour afficher des graphiques interactifs

Présentation du projet

Ce projet consiste à développer une application web permettant de lire, parser et visualiser des données CSV à l'aide de JavaScript natif. L'objectif principal est de démontrer l'utilisation de l'API fetch() pour récupérer des fichiers CSV, de parser ces données, et de les visualiser sous forme de graphiques interactifs avec la bibliothèque Chart.js.

Le projet inclut deux implémentations : une version simple qui lit et affiche les données dans la console, et une version complète qui génère un graphique en barres visualisant les températures globales moyennes de 1880 à 2024 à partir des données de la NASA.

Ce projet illustre les concepts fondamentaux du traitement de données côté client, notamment la manipulation de fichiers CSV, l'utilisation d'APIs asynchrones, et la création de visualisations de données interactives.

Fonctionnalités principales

Le projet offre plusieurs fonctionnalités essentielles pour le traitement et la visualisation de données CSV :

Lecture de fichiers CSV

Utilisation de fetch() pour récupérer des fichiers CSV depuis le serveur de manière asynchrone.

Parsing des données

Transformation des fichiers CSV texte en structures de données JavaScript exploitables (tableaux, objets).

Visualisation graphique

Création de graphiques interactifs avec Chart.js pour représenter visuellement les données.

Traitement des données

Filtrage, transformation et calcul de valeurs dérivées (comme l'ajout d'une valeur constante pour ajuster l'affichage).

Structure des fichiers CSV

Les fichiers CSV utilisés contiennent des données de température globale annuelle. Le fichier ZonAnn.Ts+dSST.csv inclut les colonnes suivantes :

  • Year : L'année de mesure
  • Glob : Température globale moyenne
  • NHem : Température hémisphère nord
  • SHem : Température hémisphère sud
  • Et d'autres colonnes pour différentes zones géographiques

Le format CSV est parsé ligne par ligne, en ignorant la première ligne (en-têtes), puis chaque ligne est divisée par les virgules pour extraire les valeurs individuelles.

Architecture

Le projet suit une architecture simple et modulaire basée sur JavaScript vanilla, sans framework lourd. L'architecture se compose de trois couches principales :

1. Couche de récupération des données

La fonction getData() utilise l'API fetch() pour récupérer le fichier CSV. Cette fonction est asynchrone et retourne une Promise, permettant une gestion élégante des opérations asynchrones. 

async function getData() {
    try {
        let response = await fetch('ZonAnn.Ts+dSST.csv');
        let data = await response.text();
        
        let table = data.split('\n');
        table = table.slice(1); // Supprimer la première ligne (en-têtes)
        
        table.forEach(ligne => {
            let colonnes = ligne.split(',');
            let annee = colonnes[0];
            let temp = colonnes[1];
            
            xLabels.push(annee);
            yLabels.push(parseFloat(temp) + 14);
        });
    }
    catch (e) {
        console.log('Erreur : ');
        console.log(e);
    }
}

2. Couche de traitement des données

Les données parsées sont stockées dans des tableaux séparés pour les labels (axes X) et les valeurs (axes Y). Les valeurs de température sont converties en nombres flottants et ajustées pour l'affichage (+14 pour normaliser l'échelle).

3. Couche de visualisation

Chart.js est utilisé pour créer un graphique en barres. La configuration inclut :

  • Type de graphique : barres
  • Labels sur l'axe X : années (1880-2024)
  • Données sur l'axe Y : températures moyennes
  • Personnalisation des couleurs et bordures
  • Formatage de l'axe Y avec le symbole °C
const ctx = document.getElementById('myChart');
new Chart(ctx, {
    type: 'bar',
    data: {
        labels: xLabels,
        datasets: [{
            label: 'Global Average Temperature ',
            data: yLabels,
            backgroundColor: 'rgba(255, 99, 132, 0.2)',
            borderColor: 'rgba(255, 99, 132, 1)',
            borderWidth: 1
        }]
    },
    options: {
        scales: {
            y: {
                min: 13,
                ticks: {
                    callback: function(value, index, ticks) {
                        return value + '°';
                    }
                }
            }
        }
    }
});

Développement

Version 1 : Lecture simple avec console.log

La première version (index1.html et index1.js) démontre les concepts de base de la lecture de fichiers CSV. Elle utilise fetch() pour récupérer le fichier, parse le contenu, et affiche les résultats dans la console du navigateur. 

async function getData(){
    try{
        let response = await fetch('test.csv');
        let data = await response.text();
        
        table = data.split('\n').slice(1);
        
        console.log(table);
        
        table.forEach(ligne => {
            let colonnes = ligne.split(',');
            let annee = colonnes[0];
            let temp = colonnes[1];
            console.log(annee, temp);
        });
    }
    catch(e){
        console.log('Erreur : ');
        console.log(e);
    }
}

getData();

Cette version est idéale pour comprendre le flux de données : récupération → parsing → traitement. Elle sert de point de départ pour les développeurs débutants souhaitant comprendre le traitement de fichiers CSV en JavaScript.

Version 2 : Visualisation complète avec Chart.js

La deuxième version (index.html et index.js) étend la fonctionnalité avec une visualisation graphique complète. Cette implémentation :

01

Récupération asynchrone

Utilise async/await pour gérer les opérations asynchrones de manière propre et lisible.

02

Stockage des données

Maintient deux tableaux globaux (xLabels et yLabels) pour séparer les labels et les valeurs.

03

Rendu graphique

Crée un graphique interactif avec Chart.js une fois les données chargées, en utilisant l'élément canvas HTML5.

Gestion des erreurs

Le code implémente une gestion d'erreurs robuste avec des blocs try/catch. Si le fichier CSV n'est pas trouvé ou s'il y a une erreur lors du parsing, un message d'erreur est affiché dans la console, permettant un débogage facile.

Optimisations et bonnes pratiques

Plusieurs bonnes pratiques sont appliquées dans ce projet :

  • Asynchrone : Utilisation d'async/await pour un code plus lisible
  • Gestion d'erreurs : Blocs try/catch pour capturer les erreurs
  • Parsing sécurisé : Conversion explicite avec parseFloat() pour éviter les erreurs de type
  • Séparation des préoccupations : Fonctions dédiées pour la récupération et l'affichage
  • Performance : Utilisation de slice(1) pour ignorer efficacement la première ligne

Technologies utilisées

Le projet utilise exclusivement des technologies web standards et une bibliothèque externe pour la visualisation :

JavaScript (ES6+)

Langage de programmation principal, utilisant les fonctionnalités modernes comme async/await, fetch(), et les fonctions fléchées.

Fetch API

API native du navigateur pour récupérer des ressources (fichiers CSV) de manière asynchrone, remplaçant l'ancien XMLHttpRequest.

Chart.js

Bibliothèque JavaScript open-source pour créer des graphiques interactifs et responsives, utilisée ici pour générer des graphiques en barres.

HTML5

Structure de la page web, avec l'élément <canvas> utilisé par Chart.js pour le rendu graphique.

Exemples de code détaillés

Parsing d'une ligne CSV

Voici comment une ligne CSV est parsée et transformée en données utilisables : 

// Exemple de ligne CSV : "1880,-.18,-.30,-.05"
table.forEach(ligne => {
    // Diviser la ligne par les virgules
    let colonnes = ligne.split(',');
    
    // Extraire les valeurs
    let annee = colonnes[0];        // "1880"
    let temp = colonnes[1];         // "-.18"
    
    // Stocker dans les tableaux
    xLabels.push(annee);                    // Pour l'axe X
    yLabels.push(parseFloat(temp) + 14);    // Pour l'axe Y (ajout de 14 pour normaliser)
});

Configuration Chart.js avancée

La configuration complète du graphique Chart.js inclut plusieurs options de personnalisation : 

options: {
    responsive: true,                    // Le graphique s'adapte à la taille du conteneur
    maintainAspectRatio: false,         // Permet de contrôler les proportions
    scales: {
        y: {
            min: 13,                     // Valeur minimale de l'axe Y
            beginAtZero: false,          // Ne commence pas à zéro pour mieux voir les variations
            ticks: {
                callback: function(value, index, ticks) {
                    return value + '°';  // Formatage personnalisé avec le symbole °
                }
            }
        },
        x: {
            ticks: {
                maxRotation: 45,         // Rotation des labels pour éviter le chevauchement
                minRotation: 0
            }
        }
    },
    plugins: {
        legend: {
            display: true,               // Afficher la légende
            position: 'top'
        },
        tooltip: {
            enabled: true                // Afficher les tooltips au survol
        }
    }
}
